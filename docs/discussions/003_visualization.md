# Discussion 003 — 可視化・UI の方向性

**ステータス**: 🔶 議論中  
**作成日**: 2025-04  
**参加者**: 開発チーム  

---

## 背景

プロトタイプの可視化はSVG + vanilla JSで実装されています。
今後の開発方針として、技術スタックとUI表現の両面で選択が必要です。

---

## 技術スタックの選択肢

### 現状: HTMLシングルファイル（vanilla JS + SVG）

```
src/index.html  ← すべてここに書いてある
```

**メリット**: 
- Gitに上げてそのままブラウザで開ける
- 依存ゼロ、ビルド不要
- 非エンジニアでも共有・確認しやすい

**デメリット**:
- コードが肥大化しやすい
- コンポーネント分割・テストがしにくい

---

### 案A: このまま vanilla JS + SVG を育てる

ファイルを複数に分割し、ローカルサーバー（`npx serve`）で動かす。

```
src/
  index.html
  js/
    state.js      ← 状態管理
    combat.js     ← 戦闘計算
    render.js     ← SVG描画
    controls.js   ← UI操作
  css/
    style.css
```

**採用条件**: 機能追加が少なく、シンプルさを優先する場合

---

### 案B: React + Vite へ移行

```
src/
  main.jsx
  components/
    BattleMap.jsx
    FacilityNode.jsx
    SetupPanel.jsx
    ScoreBar.jsx
  hooks/
    useBattleState.js
    useCombatStep.js
  lib/
    combat.js
    facilities.js
```

**メリット**: コンポーネント分割、状態管理が明快、将来の機能拡張に強い  
**デメリット**: Node.js環境が必要、ビルドステップが増える  
**採用条件**: 機能が増える見込みがある場合、複数人で開発する場合

---

## 可視化の表現に関する議論

### 現在の表現

- 施設: 円（色で占領状況を表示）
- 兵力: 縦バー（左=味方、右=敵）
- 交戦中: 黄色のリング

### 追加・改善候補

| 機能 | 優先度 | 難易度 | 備考 |
|------|--------|--------|------|
| 兵力の推移グラフ（時系列） | 高 | 中 | Chart.jsで実装可能 |
| ティックごとの結果ログ | 高 | 低 | テキストログ |
| 複数シナリオの比較 | 中 | 高 | 画面分割 or タブ切り替え |
| 施設マップのカスタム配置 | 中 | 高 | ドラッグ&ドロップ |
| モバイル対応（タッチUI） | 高 | 中 | スライダーのタッチ操作 |
| 結果のエクスポート（画像/CSV） | 低 | 中 | |
