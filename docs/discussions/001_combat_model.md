# Discussion 001 — 戦闘計算モデルの選定

**ステータス**: 🔶 暫定採用中（要議論）

**参加者**: 開発チーム  

---

## 背景

シミュレーターの核となる「戦闘でどれだけ兵力が減るか」の計算モデルを決める必要があります。
現在は暫定的にランチェスター線形則を採用していますが、実際のゲームの仕様に合わせて変更が必要になる可能性があります。

---

## 現在の実装（ランチェスター線形則）

各ティックごとに以下を計算します:

```
味方の損失 = (敵兵数 × 敵強度) ÷ UNIT_HP
敵の損失   = (味方兵数 × 味方強度) ÷ UNIT_HP
```

**例**: 味方5部隊（強度3）vs 敵5部隊（強度3）の場合
- 1ティックごとに双方 `5 × 3 ÷ 10 = 1.5` 部隊ずつ減少
- 約3〜4ティックで決着

**特徴**:
- 兵数に比例してダメージが増える（数的優位が線形に効く）
- 計算がシンプルで直感的
- パラメータが少なく調整しやすい

---

## 検討中の代替モデル

### A. ランチェスター二乗則（集中戦闘モデル）

```
味方の損失 = (敵兵数² × 敵強度) ÷ UNIT_HP
```

- 兵数の二乗に比例してダメージが増える
- 数的優位の効果がより大きい（少数で大軍に勝ちにくい）
- 近代的な組織戦闘に近いとされる

### B. ゲーム固有の計算式

実際のゲームの計算式が公開・判明している場合はそちらを優先する。
以下の情報があれば実装できます:

- [ ] 1ティック（または1秒）あたりのダメージ計算式
- [ ] ユニットのHP・攻撃力のパラメータ定義
- [ ] 防御力・属性相性などの補正係数

### C. 確率モデル（ランダム要素あり）

各ティックに乱数を加えることで、同じ配置でも結果がばらつく。
「最悪ケース / 期待値 / 最良ケース」の3パターン表示が可能になる。
