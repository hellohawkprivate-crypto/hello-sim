# Discussion 002 — 部隊の施設間移動

**ステータス**: ❌ 未実装（設計議論中）  
**作成日**: 2025-04  
**参加者**: 開発チーム  

---

## 背景

現在のプロトタイプでは、**部隊は配置された施設に固定**されています。
実際のゲームでは部隊が施設間を移動することがあるため、その仕様をどう扱うかを決める必要があります。

---

## 現状（v0）

```
セットアップ時に各施設へ配置 → シミュレーション開始 → 部隊は動かない
```

- 占領後の部隊は施設に留まる
- 後方支援・増援などの概念は存在しない

---

## 移動メカニズムの選択肢

### 案A: 移動なし（現状維持）

施設間の移動は考慮せず、「配置の最適化」のみをシミュレートする。

**メリット**: シンプル、計算が安定  
**デメリット**: ゲームの実態と乖離する可能性がある

---

### 案B: 自動進軍（施設制圧後に隣へ）

ある施設を制圧した勝利部隊が、一定ティック後に隣接する未制圧施設へ自動で進軍する。

```
施設Aを制圧 → 2ティック待機 → 隣接する敵/中立施設Bへ進軍開始
```

**実装イメージ**:
```js
// 制圧済み施設をチェックし、勝利残存部隊を隣へ送る
function autoAdvance(facs) {
  facs.forEach(f => {
    if (f.owner === 'ally' && f.enemy.count === 0 && f.waitTick === 0) {
      const targets = getAdjacentEnemyOrNeutral(f.id, facs);
      if (targets.length > 0) splitAndAdvance(f, targets[0], facs);
    }
  });
}
```

**メリット**: ゲームの動的な展開を再現できる  
**デメリット**: 実装複雑、部隊分割・合流の扱いが難しい

---

### 案C: 手動移動（ユーザーが指示）

シミュレーション中にユーザーが部隊の移動先を指定できる。
「作戦指示モード」として別フェーズで扱う。

**メリット**: 最も実態に近い作戦シミュレーション  
**デメリット**: UIが複雑になる、実装コストが高い
