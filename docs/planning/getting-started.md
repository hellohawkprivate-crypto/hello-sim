# Getting Started — 環境構築と動かし方

## 必要なもの

| ツール | 用途 | 必須？ |
|--------|------|--------|
| Git | リポジトリ管理 | ✅ 必須 |
| モダンブラウザ | 動作確認 | ✅ 必須 |

## リポジトリのクローン

```bash
git clone <リポジトリURL>
cd battle-sim
```

---

## 動かし方

### 方法A: ブラウザで直接開く（最速）

```bash
# macOS
open src/index.html

# Windows
start src/index.html

# または: ブラウザにファイルをドラッグ&ドロップ
```

> ⚠️ 将来的にファイルを分割した場合は、CORS制限でこの方法は使えなくなります。

---

### 方法B: ローカルサーバーで開く（推奨）

```bash
# Node.jsがある場合
npx serve src

# または
npx http-server src -p 3000
```

ブラウザで `http://localhost:3000` を開きます。

## 使い方（シミュレーターの操作）

### セットアップフェーズ

1. 画面下のスライダーで**各施設への味方配置数**を設定する
   - 🔵 = 味方領土（初期配置）
   - ⬜ = 中立領土（無人）
   - 🔴 = 敵領土（敵が守っている）
2. **味方ユニット強度** と **敵ユニット強度** を設定する（1〜6）
3. 「▶ シミュレーション開始」を押す

### シミュレーション中

- **施設の色**: 青=味方支配、赤=敵支配、グレー=中立
- **左のバー**: 味方の残存兵力
- **右のバー**: 敵の残存兵力
- **黄色のリング**: 現在交戦中の施設
- **数字（施設内）**: 青=味方兵数、赤=敵兵数

### 操作ボタン

| ボタン | 機能 |
|--------|------|
| ▶ 開始 | セットアップ内容でシミュレーション開始 |
| ⏸ 一時停止 | アニメーションを止める |
| ▶ 再開 | 続きから再生 |
| ↺ リセット | セットアップ画面に戻る |
| 速度 ×0.5〜×4 | アニメーション速度を変更 |

## ブランチ運用（チーム開発の場合）

```
main          ← 安定版（チームに共有するブランチ）
dev           ← 開発中の統合ブランチ
feature/XXX   ← 機能ごとの作業ブランチ
```

### 作業の流れ

```bash
# 作業開始
git checkout dev
git pull origin dev
git checkout -b feature/enemy-setup-ui

# 変更・コミット
git add .
git commit -m "feat: 敵配置をUIから変更できるように"

# プッシュ
git push origin feature/enemy-setup-ui

# プルリクエストを出して dev にマージ
```

## トラブルシューティング

### 画面が真っ白になる

- ブラウザのコンソール（F12）でエラーを確認する
- `index.html` をローカルサーバー経由で開いているか確認する

### スライダーが動かない

- ブラウザの拡張機能が干渉していないか確認する
- 別のブラウザで試す

### アニメーションが止まらない

- ↺ リセットボタンを押す
- それでも止まらない場合はページをリロードする（F5）
